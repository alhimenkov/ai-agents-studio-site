01 — Главная мысль

Агент не заменяет вкус.
Он забирает курьерскую работу.

Сейчас ты переносишь контекст между чатом, файлами и Трекером руками. Агент — это тот же ChatGPT, только он живёт в твоих файлах и умеет действовать: читать, править, ходить в интернет, дёргать сервисы. Первый вечер собирается за 30 минут по одной методичке. Дальше — вопрос вкуса, а не техники.

02 — Где мы сейчас

Схема рабочая.
Но ты в ней — курьер.

Чат

одно окно, память до конца сессии, ответ живёт только в браузере

Фреймы

промпты и заготовки в папках, но модель их сама не читает

База

бренды и правила есть — ты подмешиваешь их в каждый запрос вручную

Дальше — способы убрать тебя из роли курьера: одна папка проекта, постоянная инструкция, память, и агент, который работает прямо с файлами.
03 — Из чего собран агент

Три детали, без которых это
остаётся просто чатиком.

01

Контекст

что модель видит прямо сейчас: твой запрос, куски файлов, результаты прошлых шагов. Окно конечно — забил мусором, модель поплыла.

02

Инструменты

чем модель действует: читать и писать файлы, запускать команды в терминале, ходить в интернет, дёргать трекер. Без них — просто чатик.

03

Память

что переживает разговор: план задачи, заметки, база знаний. Это то, к чему модель возвращается между шагами.

Дальше цикл: посмотрел контекст → выбрал инструмент → сделал шаг → глянул результат → следующий. Прерываемый: остановил в любой момент, поправил курс, отпустил дальше.

04 — Workflow или агент

Не включать автономию там,
где путь и так известен.

WORKFLOW
  • Шаги прописаны в коде, модель идёт по рельсам
  • Предсказуемо, дёшево по токенам, легко чинить
  • Пример: разбор брифа по 4 шагам с проверкой
АГЕНТ
  • Модель сама решает, какие шаги и какими инструментами
  • Гибко, но дороже и менее предсказуемо
  • Пример: ресёрч, где заранее не знаешь, куда идти

Почти всё реально рабочее — workflow из пары шагов с проверкой между ними. Полную автономию включай там, где путь заранее не знаешь.

05 — Пять рабочих схем

Ими читается почти
любой AI-инструмент.

01
Цепочка

фиксированные шаги с проверкой между ними. Написал план — идёшь по плану.

02
Роутер

сначала классифицируешь вход, потом отправляешь в нужную ветку.

03
Параллель

один вход гоняешь несколькими путями сразу — либо разные куски, либо для надёжности.

04
Оркестратор

ведущий бьёт задачу на куски и раздаёт субагентам. Когда куски заранее не известны.

05
Оценщик

один генерит, второй критикует. По кругу, пока не станет хорошо.

Для студии базовый набор: цепочка для разбора брифа, оценщик для критики черновиков, параллель для ресёрча территорий.
06 — Инструменты на одной оси

Вопрос не в модели.
Вопрос — где живёт агент.

Основной

Claude Code

Терминальный агент. Настоящие субагенты с чистым контекстом, окно до миллиона токенов, скиллы под рутину.

Основной

Codex

Одна связка на терминал, IDE, облако, GitHub, Slack. Силён в долгих автономных прогонах. Заперт на моделях OpenAI.

Опция

Cursor

Форк VS Code вокруг ИИ. Выбор модели под задачу, привычный редактор. Летом 2025 были неожиданные счета — следи за расходом.

В поле зрения

Antigravity

Гугловая агент-платформа: инбокс задач, агент сам ходит в браузер и тестирует. Сырой интерфейс — не под дедлайн.

Модели меняются каждый месяц — парадигма важнее ярлыка. Начинать стоит с терминального агента: рабочие цепочки и скиллы — его родная территория.

07 — Первый вечер

За 30 минут — от установки
до сводки по своему брифу.

1
Поставить агента

Открой Terminal и введи:

npm install -g @anthropic-ai/claude-code

Нужен Node.js и подписка Claude — для старта хватит за $20/мес. Дальше при каждом запуске просто пишешь claude.

2
Открыть папку с брифом

Возьми любой обезличенный бриф. Положи его в отдельную папку — прямо на рабочем столе:

cd ~/Desktop/razbor-brifov && claude

Или перетащи папку мышкой прямо в окно Terminal — путь подставится сам.

3
Дать первую задачу

Просто скажи агенту:

Прочитай бриф в этой папке. Разбери по шагам: приёмка, противоречия, флаги, сводка на одну страницу. Ничего не пиши, пока не покажешь план.

Читаешь план, поправляешь одну строчку, отпускаешь дальше.

4
Отклонить одно, принять другое

Агент показывает каждое изменение как «было → стало». Отклоняешь одно, принимаешь другое — почувствуй, как ты контролируешь, а не только читаешь.

Не переключайся в автомат. Начинай в режиме «спрашивает на каждое действие» — Shift+Tab переключает режимы.

→
Что получишь

Сводку по брифу на одну страницу + список конкретных вопросов клиенту. За те же полчаса, что раньше уходили на первое чтение и заметки. Это твой первый маленький впечатляющий результат.

08 — Как ставить задачу агенту

Почти как подрядчику.
Только ещё жёстче.

Сначала план
«расскажи, как будешь делать, ничего не трогай». Поправить план дешевле, чем полчаса не туда.
Заставь спрашивать
«если чего-то не хватает — сначала вопросы, а не догадки». Одна фраза выключает додумывание.
Мелкими шагами
одна задача за запрос. Не «собери целиком», а «сделай шаг, покажи, дальше решу».
Новый чат
закончил задачу — новая сессия. Старый контекст подмешивается в решения.
Проси доказательство
«покажи, чем ты это проверил»: вывод команды, кусок результата, конкретный файл.
Отправь в разведку
«сходи в интернет, дай ссылки на первоисточники». Когда нужны свежие данные или чужие продукты.
Диктуй, не печатай
длинную постановку быстрее наговорить. Агент сам структурирует сырьё.

11 — План на месяц

Сначала личный инструмент.
Потом демо. Потом наружу.

Неделя 1 · 3–4 часа
Агент в руках

Поставить Claude Code, оформить подписку, пройти первые 30 минут на своей папке, включить диктовку.

Неделя 2 · вечер-два
Перенести своё

Вытащить инструкции из ChatGPT-агента, прогнать на 2–3 обезличенных брифах, завести персону внутреннего пользователя.

Неделя 3 · вечер-два
CJM + самопроверки

Разобрать путь коллеги, заполнить базу по 2–3 брендам, собрать эталонный набор брифов для авто-прогона.

Неделя 4 · выходные
Интерфейс и наружу

Один бриф от загрузки до сводки, GitHub и Render с ключом в переменных, демо для ревью по скелету «боль — показ — цифра».

12 — Правила

Что держит систему
взрослой.

— Не выдумывать. Не уверен — вопрос.
— Готово = есть проверка.
— Сначала план, потом действие.
— Секреты не лежат в проекте.
— Одна задача — одна папка.
— Внешние действия — только с подтверждением.
— Новая задача — чистый контекст.
— Мягкость к людям, жёсткость к процессу.
